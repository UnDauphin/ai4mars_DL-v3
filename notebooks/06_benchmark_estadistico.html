06 - Benchmark estadistico y resultados finales#

Este notebook consolida el benchmark final de segmentacion semantica sobre AI4MARS. Consume los artefactos producidos por la propuesta original SegFormer-B2 y los benchmarks 05a-05e y genera la comparacion cuantitativa, el ranking, los trade-offs de costo y las pruebas estadisticas exigidas por el entregable.

Entradas esperadas:

  • results/benchmark_results.csv: metricas agregadas por modelo.

  • Resultados por seed extraibles de los outputs ejecutados de los notebooks de modelos, o un archivo results/benchmark_per_seed.csv con columnas model, seed, mIoU.

Reglas experimentales usadas en todo el benchmark: mismo split train/val, mismo gold test set MSL min3, mismas seeds [42, 123, 7], misma normalizacion calculada solo con train y metricas sobre pixeles validos (ignore_index=255).

Que incluye este notebook#

  1. Validacion del CSV agregado y ordenamiento por mIoU, distinguiendo propuesta original y benchmarks.

  2. Tabla comparativa final con media, desviacion estandar, IC95, IoU por clase, parametros y tiempo.

  3. Ranking visual con intervalos de confianza.

  4. Analisis por clase, con foco especial en big_rock por ser la clase minoritaria.

  5. Trade-off rendimiento vs parametros y rendimiento vs tiempo de entrenamiento.

  6. Extraccion de resultados por seed para inferencia estadistica.

  7. Friedman global, Nemenyi post-hoc, comparaciones pareadas con normalidad y correccion de Holm.

  8. Resumen ejecutivo calculado dinamicamente desde los resultados finales.

def output_text_from_notebook(path: Path) -> str:
    nb = json.loads(path.read_text(encoding="utf-8"))
    chunks = []
    for cell in nb.get("cells", []):
        for out in cell.get("outputs", []):
            text = out.get("text")
            if isinstance(text, list):
                chunks.append("".join(text))
            elif isinstance(text, str):
                chunks.append(text)
            data = out.get("data", {})
            plain = data.get("text/plain") if isinstance(data, dict) else None
            if isinstance(plain, list):
                chunks.append("".join(plain))
            elif isinstance(plain, str):
                chunks.append(plain)
    return "\n".join(chunks)

def extract_per_seed_from_notebooks() -> pd.DataFrame:
    rows = []
    seed_pat = re.compile(r"Seed\s+(\d+)\s*\|")
    best_pat = re.compile(r"Mejor val mIoU=([0-9.]+)\s*\(epoch\s*(\d+)\)\s*\|\s*tiempo=([0-9.]+)s")
    test_pat = re.compile(r"Gold test mIoU=([0-9.]+)")

    for model, path in MODEL_NOTEBOOKS.items():
        if not path.exists():
            continue
        current_seed = None
        current_val = np.nan
        current_epoch = np.nan
        current_time = np.nan
        for line in output_text_from_notebook(path).splitlines():
            m_seed = seed_pat.search(line)
            if m_seed:
                current_seed = int(m_seed.group(1))
                current_val = np.nan
                current_epoch = np.nan
                current_time = np.nan
                continue
            m_best = best_pat.search(line)
            if m_best and current_seed is not None:
                current_val = float(m_best.group(1))
                current_epoch = int(m_best.group(2))
                current_time = float(m_best.group(3))
                continue
            m_test = test_pat.search(line)
            if m_test and current_seed is not None:
                rows.append({
                    "model": model,
                    "seed": current_seed,
                    "mIoU": float(m_test.group(1)),
                    "best_val_mIoU": current_val,
                    "best_epoch": current_epoch,
                    "train_time_s": current_time,
                    "source": path.name,
                })
    return pd.DataFrame(rows)

def load_per_seed_results() -> pd.DataFrame:
    extracted = extract_per_seed_from_notebooks()
    frames = []
    if not extracted.empty:
        frames.append(extracted)

    if PER_SEED_CSV.exists():
        csv_seed = pd.read_csv(PER_SEED_CSV)
        required = {"model", "seed", "mIoU"}
        missing = required - set(csv_seed.columns)
        if missing:
            raise ValueError(f"Faltan columnas en benchmark_per_seed.csv: {sorted(missing)}")
        if "source" not in csv_seed.columns:
            csv_seed["source"] = "benchmark_per_seed.csv"
        frames.append(csv_seed)

    if not frames:
        raise FileNotFoundError(
            "No se encontraron resultados por seed. Ejecuta los notebooks de modelos conservando outputs "
            "o crea results/benchmark_per_seed.csv con columnas model, seed, mIoU."
        )

    per_seed = pd.concat(frames, ignore_index=True, sort=False)
    per_seed["seed"] = per_seed["seed"].astype(int)
    # Si existe CSV manual, sus filas quedan al final y prevalecen sobre outputs embebidos.
    return per_seed.sort_values(["model", "seed", "source"]).drop_duplicates(["model", "seed"], keep="last")

def holm_adjust(pvalues: list[float]) -> np.ndarray:
    pvalues = np.asarray(pvalues, dtype=float)
    order = np.argsort(pvalues)
    adjusted_sorted = np.empty_like(pvalues)
    m = len(pvalues)
    running_max = 0.0
    for rank, idx in enumerate(order):
        adj = min(1.0, pvalues[idx] * (m - rank))
        running_max = max(running_max, adj)
        adjusted_sorted[rank] = running_max
    adjusted = np.empty_like(pvalues)
    adjusted[order] = adjusted_sorted
    return adjusted

Discusion critica#

El benchmark muestra que no existe un ganador absoluto en todas las dimensiones evaluadas. TerSeg obtiene el mayor mIoU promedio en el gold test (0.8381), pero la diferencia frente a MarsSeg (0.8339) y SegFormer-B2 (0.8337) es muy pequeña: menos de medio punto porcentual de mIoU. En terminos practicos, estos tres modelos forman un grupo superior con rendimiento equivalente dentro de la incertidumbre del experimento.

La propuesta original del proyecto, SegFormer-B2, no lidera el ranking bruto de mIoU por una diferencia minima, pero si ofrece el perfil mas solido cuando se considera el balance completo entre rendimiento, estabilidad y costo. Su desviacion estandar entre seeds es la menor del benchmark (0.0063), alcanza la mayor pixel accuracy reportada (0.9814) y queda a solo 0.0044 mIoU del lider. Ademas, dentro de los modelos ubicados a menos de un punto de mIoU del mejor resultado, es el que presenta el mejor balance entre precision y complejidad computacional. Por eso, la lectura mas defendible no es que SegFormer-B2 gane todas las metricas, sino que constituye la alternativa mas estable y eficiente dentro del grupo de alto rendimiento.

Las pruebas estadisticas refuerzan esa interpretacion cautelosa. La prueba global de Friedman no rechaza la hipotesis nula (chi2=7.1905, p=0.2069), por lo que no hay evidencia estadistica suficiente para afirmar diferencias globales robustas entre los modelos con solo tres seeds. En las comparaciones pareadas, ninguna diferencia sobrevive a la correccion de Holm (0/15 pares significativos). Aunque las diferencias contra IC-TransUNet son grandes en magnitud, el numero reducido de seeds limita la potencia estadistica y obliga a reportarlas como evidencia practica, no como conclusion inferencial definitiva.

La clase minoritaria big_rock cambia la lectura del ranking. MarsSeg obtiene el mejor IoU en esta clase (0.5219), seguido por TerSeg (0.5097), mientras que SegFormer-B2 alcanza 0.4592. Esto indica que los modelos con mecanismos mas orientados a detalle local o a manejo explicito del desbalance pueden recuperar mejor objetos pequenos y poco frecuentes. Sin embargo, big_rock sigue siendo la clase mas dificil del benchmark en promedio (IoU medio=0.4088), lo que confirma que el buen rendimiento global puede ocultar errores importantes en categorias operacionalmente relevantes.

IC-TransUNet funciona como un contraste util dentro del estudio. A pesar de incorporar una arquitectura CNN-Transformer dual branch mas compleja en concepto, su mIoU promedio recuperado desde checkpoints es 0.6986, claramente por debajo del resto de modelos. Este resultado sugiere que aumentar la complejidad arquitectonica no garantiza mejor transferencia al dominio AI4MARS, especialmente cuando el entrenamiento, la resolucion, el desbalance de clases y la adaptacion al dataset no estan tan afinados como en modelos mas compactos o mejor ajustados al protocolo.

En conjunto, los resultados sostienen una conclusion matizada: TerSeg reporta el mayor mIoU promedio, MarsSeg destaca en la clase minoritaria big_rock, y SegFormer-B2 es la propuesta original con mejor equilibrio general entre desempeno, estabilidad y costo. Las limitaciones principales son el bajo numero de seeds, el uso de un unico gold test MSL, la ausencia de validacion multi-mision y la evaluacion de DepthFormer en variante RGB sin mapas de profundidad reales. Por tanto, las conclusiones deben presentarse como evidencia experimental reproducible dentro de este protocolo, no como una superioridad universal fuera de AI4MARS/MSL.

Como trabajo futuro, se recomienda ampliar el numero de seeds para aumentar la potencia estadistica, validar los modelos en otras misiones o camaras del dataset AI4MARS, incorporar metricas adicionales sensibles a bordes y objetos pequenos, y explorar estrategias especificas para big_rock, como perdidas focal/dice mejor calibradas, muestreo dirigido por presencia de clase minoritaria, mayor resolucion de entrada o post-procesamiento morfologico. Tambien seria deseable evaluar una version de DepthFormer con mapas de profundidad reales para comprobar si su ventaja teorica aparece cuando se usa la modalidad para la que fue disenado.
